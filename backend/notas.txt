3..INICIALIZAR EL el Backend

# a. Crear la carpeta para el servidor e ingresar a ella
mkdir backend
cd backend

# b. Inicializar el proyecto de Node.js (creará el package.json)
npm init -y

# b. Instalar Express, el conector de MySQL y las herramientas necesarias
npm install express mysql2 dotenv cors

# c. Instalar nodemon como dependencia de desarrollo (para que el servidor se reinicie solo al guardar)
npm install -D nodemon

4... Configurar las Variables de Entorno (.env)
5.. Crear las tablas en phpMyAdmin

6.. nstalar la herramienta para proteger contraseñas

npm install bcryptjs 

Luego de instalarlo, puedes volver a encender el servidor con npm run dev.

¿Cómo lo probamos en Thunder Client ahora que pide Token?

Ve a la pestaña del Login en Thunder Client, copia todo ese texto largo que salió en token (sin las comillas).

Abre un New Request en Thunder Client.

Configúralo en método POST con la URL: http://localhost:5000/api/tareas

Ve a la pestaña Auth (que está al lado de Headers y Body).

Selecciona la opción Bearer Token y pega ahí el token larguísimo que copiaste.

Ahora ve a la pestaña Body -> JSON y escribe los datos de tu primera tarea:

JSON
{
  "titulo": "Terminar el backend de ADSO",
  "descripcion": "Crear los endpoints del CRUD de tareas y probarlos",
  "fecha_vencimiento": "2026-06-30"
}
